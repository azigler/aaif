## The commitment

- **At least one public, project-based contribution per month.** Non-negotiable
  floor.
- It does **not** have to be a major talk/course/workshop. A useful social
  thread, community help, short video, blog post, tutorial, or project
  contribution all count.
- The stated goal: **"sustained, meaningful engagement … a steady stream of
  helpful developer education and project participation."** (This is exactly
  Andrew's stated aim — exceed the floor, consistently.)
- Also expected: represent AAIF professionally; participate in Ambassador
  communications; share community feedback upstream.

## The three program goals (every submission should serve one)

1. **Awareness** — help devs understand AAIF projects & where they fit. (e.g. a
   thread explaining what MCP enables; a meetup talk introducing goose.)
2. **Activation** — practical resources that help devs *try* it. (getting-started
   tutorial; hands-on livestream; workshop; demo repo.)
3. **Contribution** — clearer pathways for devs to *contribute*. (first-timer
   guide; a docs/examples/recipes/templates/integration/code PR; community
   support that unblocks someone.)

## The four submission targets (projects)

`agentgateway` · `AGENTS.md` · `goose` · `MCP` · "Other AAIF Project (added
often)". One contribution may cover multiple projects — select all that apply.
**Multiple projects help reporting/promotion but do NOT increase point value.**

## What counts as a contribution

Blog posts · tutorials · technical videos · social threads · meetup talks ·
conference talks · workshops · courses · livestreams · podcast appearances ·
community help (Discord/Slack/forums/GitHub Discussions) · project
contributions (PRs, docs, examples, templates, integrations, recipes, issue
work).

### What makes a STRONG contribution

Useful to developers; **practical, grounded, technically credible — NOT polished
corporate marketing.** A strong contribution does ≥1 of: explains a real dev
problem · shows how an AAIF project solves it · helps someone get started ·
teaches a concept clearly · demonstrates a workflow · shares lessons from real
usage · makes it easier for someone else to contribute · supports another dev ·
improves a project directly.

> *"The best contributions are specific. 'Agents are the future' is not enough.
> 'Here is how I used Goose to automate a repo maintenance workflow' is much more
> useful."*

### Contribution requirements (qualifying criteria)

- Ties directly to an AAIF project.
- Helps developers understand / try / use / contribute to that project.
- **Public** (unless AAIF explicitly allows the type to be private/semi-private).
- Includes **enough evidence for review**.
- Represents your own work, or clearly explains your role.
- Follows the spirit of open, respectful, accurate engagement.

### What usually does NOT qualify

Generic AI posts not explaining an AAIF project · private convos w/o reviewable
evidence · reposts w/o original commentary · promo posts w/o
technical/educational substance · content mentioning AAIF only in passing ·
unclear role · **AI-generated content not meaningfully reviewed/tested/improved
by you** · inaccurate/misleading/harmful content.

> ⭐ **The doctrine that governs this whole harness:** *"We like agents. We do
> not want agent slop. Use agents to accelerate your work, but make sure the
> final contribution is accurate, useful, and something you stand behind."* and
> *"Agents are collaborators, not accountability shields."* → Every submission
> passes a human-verification gate before it ships. See `../../CLAUDE.md`.

## How to submit (the workflow)

Submissions go through the **AAIF Ambassador submissions repository** (Andrew is
added to it after joining — get the repo URL from the Ambassador Discord /
onboarding).

1. **Open a new issue** using the **"Ambassador Contribution Submission"** template.
2. **Fill required fields:** GitHub handle (`azigler`) · the **contribution URL**
   · related AAIF project(s) · optional notes (context for the reviewer —
   *"help the agent help you"*).
3. **Submit.** An **AI review agent** reads the artifact, classifies the type,
   checks project relevance, evaluates developer usefulness, **recommends a
   score**, and flags anything needing more info. A human may adjust.
4. **Respond in-thread** if more info is requested (your role in a collab; proof
   of event participation; recording/slide link; a better URL; project
   clarification; screenshots/summaries for community help).

Public contribution URL types accepted: blog post · GitHub PR · GitHub issue ·
YouTube video · livestream recording · conference session page · meetup event
page · slide deck · social thread · podcast episode · tutorial repo.

## Points & scoring (the "top ambassador" lever)

Contributions are weighted by **effort, usefulness, evidence**. Points feed
activity levels, **leaderboards**, reports, recognition, and badges. **You don't
calculate your own points** — the review process scores it.

| Contribution type | Points |
|---|---|
| Social thread | 5 |
| Community help (Discord/Slack) | 5–15 |
| Blog post | 15 |
| Short technical video | 15 |
| Tutorial | 20 |
| Meetup talk | 20 |
| Podcast guest | 20 |
| Livestream | 25 |
| Organizing a meetup | 25 |
| Conference talk | 30 |
| Workshop | 35 |
| **Course** | **50** |
| Project contribution | 5–50 |

Scoring also considers: contribution type · AAIF project relevance · developer
usefulness · technical depth · quality & clarity · evidence provided · public &
verifiable · bonuses/limitations. **These are baselines** — quality/depth move
the final score.

### Scorecards

Approved contributions become **scorecards** in the submissions repo — the
**source of truth** for approved points. Used to generate Ambassador
scoreboards, monthly reports, recognition summaries, promotion roundups, program
activity analysis. Each records when the contribution happened, when AAIF
approved it, and which monthly leaderboard/report includes it.

> ⏱️ **Timing rule:** late submissions still count toward **annual** totals, but
> published **monthly** leaderboards are NOT recalculated. **Submit promptly** if
> you want it in the current month's recognition.

## Recognition, status & badges

- Contribution points → **status & badges** (intended to make participation
  visible/fun/motivating; help AAIF spot active contributors for amplification,
  leadership opportunities, events, special recognition). "Not a grind."
- Badges are issued via **Credly** (see `refs/brand/ambassador-credly-badge.png`).

## Perks (tied to active participation)

Private Ambassador Discord channel · regular briefings with AAIF leadership &
maintainers · early visibility into roadmap/messaging · amplification (AAIF
social, newsletter, community) · recognition on the AAIF Ambassador page ·
exclusive swag · **VIP conference** (AGNTCon/MCPCon — discounted tickets, VIP

ribbons).

## Renewal

Reviewed **every six months**. Considers: consistency of participation · quality
· project relevance · community impact · professionalism · responsiveness ·
alignment with AAIF values · continued interest.

## Conduct

Respectful · inclusive · technically honest · helpful · supportive of
maintainers · open to feedback · clear about uncertainty · aligned with
open-source values. *"If you do not know the answer, say so. Then help find the
answer. That is much better than confidently inventing one."*

## Tips for a great first month (AAIF's own advice)

Pick **one simple** contribution. Ideas: a short "getting started" post · a
5-min walkthrough of something you learned · a technical thread explaining one
concept · help someone in the community + submit the interaction · a small
example repo · a docs improvement · a short local meetup talk · a "what I wish I
knew before trying this project" writeup.

> *"Start practical. You do not need to create the definitive guide to agentic AI
> in your first contribution. Please do not try."*
